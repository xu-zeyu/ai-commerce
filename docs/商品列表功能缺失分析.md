# 商品列表功能缺失分析

> 分析时间：2026-07-01
> 分析范围：jinHan-shop 商品模块（SPU + SPU图片）

---

## 一、当前已实现功能

| 功能模块 | 接口 | 状态 |
|---------|------|------|
| 商品SPU新增 | `POST /product/create` | ✅ 已实现 |
| 商品SPU编辑 | `PUT /product/{id}` | ✅ 已实现 |
| 商品SPU删除（逻辑删除） | `DELETE /product/{id}` | ✅ 已实现 |
| 商品SPU分页查询 | `GET /product/page` | ✅ 已实现 |
| 商品图片新增 | `POST /product/image/create` | ✅ 已实现 |
| 商品图片编辑 | `PUT /product/image/{id}` | ✅ 已实现 |
| 商品图片删除 | `DELETE /product/image/{id}` | ✅ 已实现 |
| 商品图片分页查询 | `GET /product/image/page` | ✅ 已实现 |

**已支持的筛选条件：**
- 商品名称（模糊搜索）
- SPU编码（模糊搜索）
- 供应商ID
- 分类ID
- 品牌ID
- 销售状态（上架/下架）
- 审核状态（待审核/审核通过/审核拒绝）

**已关联的维度数据：**
- 供应商名称（JOIN supplier）
- 分类名称（JOIN goods_category）
- 品牌名称（JOIN goods_brand）
- 主图（JOIN product_spu_image，image_type=1）

## 二、缺失功能清单

### 2.1 核心商品管理功能

| 优先级 | 功能 | 说明 | 影响范围 |
|-------|------|------|---------|
| 🔴 P0 | **商品详情查询** | 缺少根据ID查询商品完整信息的接口（`GET /product/{id}`），前端编辑页、详情页无法回显数据 | Controller + Handler |
| 🔴 P0 | **SKU管理** | 无SKU（规格）模块，无法管理商品的颜色、尺寸、价格、库存等变体信息 | 新增模块 `product_sku` |
| 🔴 P0 | **价格管理** | SPU实体无价格字段，缺少售价、成本价、原价等价格体系 | Entity + Command |
| 🔴 P0 | **库存管理** | SPU实体无库存字段，SKU维度的库存管理完全缺失 | Entity + Handler |
| 🟡 P1 | **商品审核功能** | `AuditStatusEnum` 已定义但无审核接口，缺少审核通过/拒绝的处理器 | Handler + Controller |
| 🟡 P1 | **商品上下架** | `SaleStatusEnum` 已定义但无独立的上下架接口，当前只能通过编辑接口修改 | Handler + Controller |
| 🟡 P1 | **批量操作** | 缺少批量删除、批量上架、批量下架、批量修改分类等批量操作接口 | Controller |
| 🟡 P1 | **商品详情编辑** | 编辑接口 `ProductSpuCommand` 字段有限，缺少富文本详情、规格参数等字段 | Command + Handler |

### 2.2 商品属性与规格

| 优先级 | 功能 | 说明 |
|-------|------|------|
| 🟡 P1 | **商品规格管理** | 无规格模板（如颜色、尺寸、材质），无法定义SPU的规格属性 |
| 🟡 P1 | **商品属性管理** | 无自定义属性（如产地、保质期、重量），无法扩展商品信息 |
| 🟢 P2 | **规格值管理** | 规格值的CRUD（如：红色、蓝色、XL、XXL） |
| 🟢 P2 | **属性分组** | 属性按组管理（基础信息、规格参数、包装信息） |

### 2.3 商品分类与标签

| 优先级 | 功能 | 说明 |
|-------|------|------|
| 🟡 P1 | **多级分类支持** | 当前分类是单级关联，缺少分类树形结构和多级分类支持 |
| 🟢 P2 | **商品标签** | 无标签管理（新品、热销、推荐、限时折扣等营销标签） |
| 🟢 P2 | **商品分组** | 无商品分组/合集功能（如：套装、搭配推荐） |

### 2.4 搜索与筛选增强

| 优先级 | 功能 | 说明 |
|-------|------|------|
| 🟡 P1 | **价格区间筛选** | 分页查询缺少价格区间筛选条件 |
| 🟡 P1 | **销量排序** | 当前只支持 sort+createdTime 排序，缺少按销量、价格、创建时间等排序 |
| 🟢 P2 | **高级搜索** | 缺少 Elasticsearch 等全文搜索支持，模糊搜索性能随数据量下降 |
| 🟢 P2 | **创建时间范围筛选** | 分页查询缺少时间范围筛选 |

### 2.5 图片与媒体管理

| 优先级 | 功能 | 说明 |
|-------|------|------|
| 🟡 P1 | **图片类型枚举化** | `imageType` 使用 Integer，应改为枚举（MAIN=1, CAROUSEL=2, DETAIL=3） |
| 🟡 P1 | **图片排序** | 同类型多张图片缺少排序字段 |
| 🟢 P2 | **视频管理** | 无商品视频上传和管理功能 |
| 🟢 P2 | **图片裁剪/压缩** | 上传图片后缺少自动裁剪、压缩处理 |

### 2.6 数据安全与审计

| 优先级 | 功能 | 说明 |
|-------|------|------|
| 🟡 P1 | **操作日志** | 虽有 `@Log` 注解，但缺少商品变更的详细日志（字段级变更记录） |
| 🟡 P1 | **版本控制** | 无商品版本管理，无法回滚到历史版本 |
| 🟢 P2 | **数据校验** | 分类ID、品牌ID、供应商ID 缺少存在性校验（当前只校验 SPU编码唯一性） |

### 2.7 导入导出

| 优先级 | 功能 | 说明 |
|-------|------|------|
| 🟢 P2 | **商品导出** | 缺少 Excel/CSV 导出功能 |
| 🟢 P2 | **商品导入** | 缺少 Excel 批量导入功能 |
| 🟢 P2 | **导入模板下载** | 缺少导入模板下载接口 |

### 2.8 前端交互增强

| 优先级 | 功能 | 说明 |
|-------|------|------|
| 🟡 P1 | **商品预览** | 缺少商品在用户端的预览功能 |
| 🟢 P2 | **商品复制** | 缺少复制商品功能（基于现有商品快速创建新品） |
| 🟢 P2 | **商品排序拖拽** | 前端缺少拖拽排序交互 |
| 🟢 P2 | **快捷操作** | 列表页缺少快捷上架/下架按钮 |

## 三、数据模型缺失

### 3.1 SKU 模型（建议新增）

```java
@Data
@TableName("product_sku")
public class ProductSku {
    @TableId(type = IdType.AUTO)
    private Long id;
    private Long spuId;                    // SPU ID
    private String skuCode;                // SKU编码
    private String specInfo;               // 规格信息JSON（如：{"颜色":"红色","尺寸":"XL"}）
    private BigDecimal price;              // 售价
    private BigDecimal originalPrice;      // 原价
    private BigDecimal costPrice;          // 成本价
    private Integer stock;                 // 库存数量
    private Integer lockedStock;           // 锁定库存
    private Integer salesCount;            // 销量
    private String barcode;                // 条形码
    private Integer status;                // 状态：0-禁用 1-启用
    @TableLogic
    private Long deleted;
    private Long createdBy;
    private Long updatedBy;
    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createdTime;
    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updatedTime;
}
```

### 3.2 商品规格属性模型（建议新增）

```java
@Data
@TableName("product_spec")
public class ProductSpec {
    @TableId(type = IdType.AUTO)
    private Long id;
    private String name;                   // 规格名称（如：颜色、尺寸）
    private Integer sort;
    @TableLogic
    private Long deleted;
    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createdTime;
    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updatedTime;
}

@Data
@TableName("product_spec_value")
public class ProductSpecValue {
    @TableId(type = IdType.AUTO)
    private Long id;
    private Long specId;                   // 规格ID
    private String value;                  // 规格值（如：红色、XL）
    private Integer sort;
    @TableLogic
    private Long deleted;
    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createdTime;
}
```

### 3.3 SPU 字段补充建议

在现有 `ProductSpu` 实体中建议新增字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `price` | BigDecimal | 最低售价（冗余，便于列表展示） |
| `originalPrice` | BigDecimal | 原价 |
| `stock` | Integer | 总库存（冗余，SKU库存汇总） |
| `detailHtml` | String | 商品详情（富文本） |
| `unit` | String | 销售单位（件、套、箱） |
| `weight` | BigDecimal | 重量（kg） |
| `volume` | BigDecimal | 体积（m³） |
| `keywords` | String | 搜索关键词 |
| `tag` | String | 标签JSON数组 |

## 五、代码规范问题

| 问题 | 位置 | 建议 |
|------|------|------|
| `ProductSpuPageResponse` 直接继承 QueryResult | response 包 | 应独立定义 Response 字段，避免暴露内部查询模型 |
| `imageType` 使用 Integer 而非枚举 | ProductSpuImage | 新增 `ImageTypeEnum` |
| 缺少分类/品牌/供应商存在性校验 | ProductSpuCreateHandler | create 时校验关联数据是否存在 |
| 删除无级联处理 | ProductSpuDeleteHandler | 删除 SPU 应同时处理关联的图片、SKU 等数据 |
