# 测试服务器部署规范

## 目标

本部署方案面向测试服务器。服务器上的 `nginx`、`mysql`、`redis`、`rabbitmq`、`minio` 按公共基础设施管理，不绑定某一个后端项目，不添加业务前缀；当前项目只维护自己的应用编排、配置仓库、应用配置、应用数据和应用日志。

核心原则：

- 公共基础设施配置放在 `/opt/docker/{service}`，例如 `/opt/docker/nginx`、`/opt/docker/mysql`、`/opt/docker/redis`。
- 公共基础设施数据放在 `/data/docker/{service}`，例如 `/data/docker/mysql`、`/data/docker/redis`。
- 当前项目部署编排放在 `/opt/jinhan/ai-commerce-test`。
- 当前项目配置仓库放在 `/opt/jinhan/ai-commerce-test/config-repo`。
- 当前项目应用数据和日志单独放在 `/data/jinhan/ai-commerce-test` 与 `/var/log/jinhan/ai-commerce-test`。
- 后续新增后端项目时复用同一套 `nginx/mysql/redis/rabbitmq/minio`，只新增应用服务、配置文件、数据库 schema 和 Nginx 路由。

## 命名规范

| 类型 | 命名 |
| --- | --- |
| Compose 项目名 | `jinhan-commerce-test` |
| 公共基础设施服务名 | `nginx`、`mysql`、`redis`、`rabbitmq`、`minio` |
| 公共基础设施容器名 | `nginx`、`mysql`、`redis`、`rabbitmq`、`minio` |
| 当前项目服务名 | `config-server`、`admin-web` |
| 当前项目容器名 | `config-server`、`jinhan-shop-admin-web` |
| Docker 网络 | `jinhan-commerce-test-edge`、`jinhan-commerce-test-backend` |

## 目录职责

| 路径 | 职责 |
| --- | --- |
| `/opt/docker/nginx` | 服务器公共 Nginx 配置、证书和日志 |
| `/opt/docker/mysql` | 服务器公共 MySQL 配置和日志 |
| `/opt/docker/redis` | 服务器公共 Redis 配置 |
| `/data/docker/mysql` | 服务器公共 MySQL 数据 |
| `/data/docker/redis` | 服务器公共 Redis 数据 |
| `/data/docker/rabbitmq` | 服务器公共 RabbitMQ 数据 |
| `/data/docker/minio` | 服务器公共 MinIO 数据 |
| `/opt/jinhan/ai-commerce-test` | 当前项目测试环境部署编排 |
| `/opt/jinhan/ai-commerce-test/config-repo` | 当前项目 Spring Cloud Config 配置仓库 |
| `/opt/jinhan/ai-commerce-test/admin-web/config` | 当前项目后台服务外部配置 |
| `/data/jinhan/ai-commerce-test/admin-web` | 当前项目后台服务业务数据 |
| `/var/log/jinhan/ai-commerce-test/admin-web` | 当前项目后台服务日志 |
| `/opt/html/jinhan-shop-web-test` | 当前项目前端静态资源 |

## 端口策略

| 服务 | 容器端口 | 宿主机默认端口 | 暴露策略 |
| --- | ---: | ---: | --- |
| Nginx HTTP | 80 | 80 | 对外开放 |
| Nginx HTTPS | 443 | 443 | 对外开放 |
| Admin Web | 8081 | 8081 | 仅绑定 `127.0.0.1` |
| MySQL | 3306 | 33306 | 对外开放 |
| Redis | 6379 | 6379 | 对外开放 |
| RabbitMQ AMQP | 5672 | 5672 | 仅绑定 `127.0.0.1` |
| RabbitMQ 管理台 | 15672 | 15672 | 仅绑定 `127.0.0.1` |
| MinIO API | 9000 | 9000 | 仅绑定 `127.0.0.1` |
| MinIO Console | 9001 | 9001 | 仅绑定 `127.0.0.1` |

## 后续新增后端项目

新增后端项目时，不新增 `mysql`、`redis`、`nginx` 容器，只做以下增量：

- 在 MySQL 中新增数据库 schema 和应用账号。
- 在 `/opt/jinhan/ai-commerce-test/config-repo` 新增该服务的 `{application}-test.yml`。
- 在 `docker-compose.yml` 新增一个应用服务。
- 在 `/opt/docker/nginx/conf/conf.d` 新增 upstream 或 location 路由。
- 为该应用单独创建 `/data/jinhan/ai-commerce-test/{service}` 和 `/var/log/jinhan/ai-commerce-test/{service}`。

## 使用方式

### 1. 本地准备发布配置

首次发布前，在本地仓库准备 `docker/production/.env`：

```sh
cd /Users/xuzeyu/Documents/company/jinHan/ai-commerce/docker/production
cp .env.example .env
```

至少确认以下变量：

```properties
IMAGE_REPOSITORY=crpi-xxx.cn-guangzhou.personal.cr.aliyuncs.com/namespace/repository
IMAGE_VERSION=test
BUILD_PLATFORM=linux/amd64
PUBLISH_MODE=buildx-push
REGISTRY_USERNAME=你的镜像仓库账号
REGISTRY_PASSWORD=你的镜像仓库密码
```

如果已经提前执行过 `docker login`，`REGISTRY_USERNAME` 和 `REGISTRY_PASSWORD` 可以留空。

注意：`.env` 中的 `DB_HOST=mysql`、`REDIS_HOST=redis`、`RABBITMQ_HOST=rabbitmq` 是 Docker Compose 内部服务名，供后端容器访问基础设施使用，不要改成服务器公网 IP。外部客户端连接 MySQL、Redis 时才使用服务器 IP，例如 `mysql -h 服务器IP -P 33306`、`redis-cli -h 服务器IP -p 6379`。

### 2. 本地打包并推送镜像

在本地执行：

```sh
cd /Users/xuzeyu/Documents/company/jinHan/ai-commerce
./docker/production/publish.sh test
```

该脚本会完成以下动作：

- 执行 Gradle `bootJar`，生成 `config-server.jar` 和 `admin-web.jar`。
- 复制 Jar 到 `docker/app/dist/`。
- 构建并推送镜像：
  - `${IMAGE_REPOSITORY}:config-server-test`
  - `${IMAGE_REPOSITORY}:admin-web-test`
- 生成 `docker/production/.release.env`，记录本次发布的镜像地址。

### 3. 上传部署脚本到服务器

服务器只需要 `docker/production` 目录下的部署文件：

```sh
scp -r docker/production root@服务器IP:/tmp/ai-commerce-production
```

如果服务器已经有这些文件，也可以只同步变更后的：

```sh
scp docker/production/docker-compose.yml root@服务器IP:/tmp/ai-commerce-production/
scp docker/production/bootstrap-test-server.sh root@服务器IP:/tmp/ai-commerce-production/
```

### 4. 服务器生成配置目录

登录服务器后执行：

```sh
ssh root@服务器IP
cd /tmp/ai-commerce-production

IMAGE_REPOSITORY=crpi-xxx.cn-guangzhou.personal.cr.aliyuncs.com/namespace/repository \
IMAGE_VERSION=test \
sh bootstrap-test-server.sh
```

脚本会生成：

- `/opt/docker/nginx`
- `/opt/docker/mysql`
- `/opt/docker/redis`
- `/data/docker/mysql`
- `/data/docker/redis`
- `/data/docker/rabbitmq`
- `/data/docker/minio`
- `/opt/jinhan/ai-commerce-test`
- `/opt/jinhan/ai-commerce-test/config-repo`
- `/opt/jinhan/ai-commerce-test/admin-web/config`

重复执行脚本时默认不会覆盖已有文件；如需重新生成：

```sh
FORCE=1 IMAGE_REPOSITORY=crpi-xxx.cn-guangzhou.personal.cr.aliyuncs.com/namespace/repository \
IMAGE_VERSION=test \
sh bootstrap-test-server.sh
```

### 5. 服务器登录镜像仓库

如果镜像仓库是私有仓库，先在服务器登录：

```sh
docker login crpi-xxx.cn-guangzhou.personal.cr.aliyuncs.com
```

### 6. 服务器拉取镜像并运行

执行生成的部署脚本：

```sh
cd /opt/jinhan/ai-commerce-test
sh scripts/deploy.sh
```

等价于手动执行：

```sh
cd /opt/jinhan/ai-commerce-test
docker compose --env-file .env -f docker-compose.yml pull
docker compose --env-file .env -f docker-compose.yml up -d
docker compose --env-file .env -f docker-compose.yml ps
```

### 7. 常用检查命令

```sh
docker compose --env-file /opt/jinhan/ai-commerce-test/.env -f /opt/jinhan/ai-commerce-test/docker-compose.yml ps
docker logs -f config-server
docker logs -f jinhan-shop-admin-web
docker logs -f nginx
```

测试外部端口：

```sh
curl http://服务器IP/
curl http://127.0.0.1:8081/actuator/health
mysql -h 服务器IP -P 33306 -u jinhan_app -p
redis-cli -h 服务器IP -p 6379 -a '你的Redis密码' ping
```
