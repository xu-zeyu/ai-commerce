# 服务器部署说明文档

> 服务器：`47.107.124.89:10745`（SSH，root）
> 系统：Alibaba Cloud Linux 3 (x86_64)
> 整体架构：**Docker 容器化部署**，前端走静态资源（Nginx），后端走 Spring Boot 微服务，统一由宿主机 Nginx 反向代理。
> 文档生成日期：2026-05-29

---

## 一、整体架构总览

```
                       公网 80 / 443
                            │
                ┌───────────▼─────────────┐
                │  宿主机 Nginx 容器        │  ← 总入口（容器名: nginx）
                │  /opt/docker/nginx       │     按域名分发
                └───────┬──────────┬───────┘
            静态前端     │          │   /api 反代
        /opt/html/*  ◄──┘          └──► 各后端容器 (Spring Boot :8080)
                                         saas-1 / ams-1 / oa-1 / pm-1 / auth-2 / sly-1
                                         oa-admin (Next.js :3000)
                                              │
                            ┌─────────────────┼──────────────────┐
                       wg-mysql:33306    wg-redis:6379      wg-minio:9000
                       (MySQL 8.0.26)  (Redis 7.4.2)      (对象存储)
```

- **所有容器** 通过外部网络 `bin_zxj-network`（172.19.0.0/16）互通，应用容器之间用**容器名**直接访问（如 `wg-mysql`、`saas-1`）。
- 业务后端容器**不对外暴露端口**，只能通过 Nginx 反向代理访问。
- 中间件（MySQL/Redis/MinIO）由 `/root/wg-app/docker-compose.yml` 统一编排，业务模块各自独立 compose。

## 二、Nginx 反向代理（流量入口）

总入口容器：`nginx`（监听宿主机 **80 / 443**）

| 路径 | 说明 |
|------|------|
| `/opt/docker/nginx/conf/nginx.conf` | 主配置，`include /opt/include/*.conf` 引入所有站点 |
| `/opt/docker/nginx/conf/include/*.conf` | **每个域名一个 vhost 配置文件**（核心路由） |
| `/opt/docker/nginx/https/` | SSL 证书目录（如 `_.wgit123.com.pem/.key`） |
| `/opt/html/` | **所有前端静态资源根目录** |
| `/opt/docker/nginx/logs/` | 访问/错误日志 |

> 还有一个 `wg-nginx` 容器（端口 18080/18443），是 wg-app 早期自带的网关，配置在 `/root/wg-app/config/nginx/`，目前 unhealthy，非主入口。

### 路由规则（统一约定）

- **前端**：`域名/` → 静态目录（`root /opt/html/xxx;` + `try_files ... /index.html` 支持前端路由）
- **后端**：`域名/api/...` → `rewrite` 去掉 `/api` 前缀 → `proxy_pass http://<upstream>`
- 静态资源 `js/css/png/jpg` 强缓存一年，`html` 不缓存（保证发版即时生效）。

### 域名 → 服务对应表（在用）

| 域名 | 类型 | 指向 |
|------|------|------|
| `ams-test.wgit123.com` | 前端(H5,HTTPS) | `/opt/html/ams-h5` + `/api`→`ams-1:8080` |
| `ams-api-test.wgit123.com` | 后端API | `ams-1:8080` |
| `ams-mng-test.wgit123.com` | 管理后台(HTTPS) | `→172.18.255.128:3000` + `/api`→`ams-1` |
| `test.saas.wgit.zxj9999.com` | 前端 | `/opt/html/saas-platform` + `/api`→`saas-1:8080` |
| `api.test.saas.wgit.zxj9999.com` | 后端API | `saas-1:8080` |
| `test.oa.wgit.zxj9999.com` | 前端 | `oa-admin:3000`（Next.js） |
| `api.test.oa.wgit.zxj9999.com` | 后端API | `oa-1:8080` |
| `test.pm.wgit.zxj9999.com` | 前端 | `/opt/html/pm-platform` + `/api`→`pm-1:8080` |
| `api.test.pm.wgit.zxj9999.com` | 后端API | `pm-1:8080` |
| `api.test.auth.wgit.zxj9999.com` | 认证中心API | `auth-2:8080` |

## 三、后端部署（Spring Boot 微服务）

所有后端模块位于 `/root/wg-app/<模块名>/`，均使用阿里云私有镜像仓库 `registry.cn-shenzhen.aliyuncs.com/zxj9999/*`。

### 运行中的后端容器

| 容器 | 镜像 | 数据库 | 说明 |
|------|------|--------|------|
| `saas-1` | wgit-saas | wgit_db | SaaS 平台后端 |
| `ams-1` | wgit-ams | wgit_ams | AMS 模块 |
| `oa-1` | wgit-oa | wgit_oa | OA 后端 |
| `pm-1` | wgit-pm | wgit_pm | 项目管理后端 |
| `auth-2` | jetboot-auth | jetboot_auth | 统一认证中心（OAuth2 Issuer） |
| `sly-1` | wgit-sly | - | sly 模块（暴露调试端口 18051） |
| `oa-admin` | oa-admin | - | OA 管理前端（Next.js, :3000） |

### 模块目录结构（以 `/root/wg-app/saas/` 为例）

```
saas/
├── docker-compose.yml      # 定义 saas-1 / saas-2 两个实例（蓝绿）
├── .env                    # 模块环境变量（profile、域名等）
├── blue_green_deploy.sh    # 蓝绿部署脚本
├── config/app/             # 挂载到容器 /app/config（logback-spring.xml 等）
└── data/app/{logs,data}/   # 日志与数据持久化
```

- 每个模块都定义了 **两个实例**（如 `saas-1` / `saas-2`），用于**蓝绿部署**，平时只跑其中一个。
- 启动参数：`--spring.profiles.active=test`（统一 test 环境），日志配置 `logback-spring.xml`。
- 多数模块开启了远程调试 `JVM_OPTS: -agentlib:jdwp=...address=*:5005`。
- 应用配置（数据库/Redis/MinIO 连接）主要通过 **`.env` 环境变量注入**，Spring 配置文件打包在镜像内。

### 关键环境变量（各模块 `.env`，已脱敏建议）

```
MYSQL_HOST=wg-mysql      MYSQL_PORT=33306   MYSQL_USERNAME=root  MYSQL_PASSWORD=123456
REDIS_HOST=wg-redis      REDIS_PORT=6379   REDIS_PASSWORD=123456

                         MINIO_DOMAIN=test.minio.wgit.zxj9999.com

```
> 注：模块间通过 `JETBOOT_AUTH_*` 对接 jetboot-auth 认证中心（client_id/secret 各模块不同）。
> 顶层 `/root/wg-app/.env` 还集中保存了短信、微信、百度、快递100、联麓支付等第三方密钥。

## 六、部署 / 发版流程

### 后端（蓝绿部署）
每个模块目录下有 `blue_green_deploy.sh`，逻辑：

1. 读取 Nginx upstream 配置，判断当前活跃实例（如 saas-1）。
2. `docker compose pull` + `up -d` 启动**另一个实例**（saas-2）。
3. 轮询容器日志，等待出现 `Started ...Application in ... seconds`（健康检查）。
4. `sed` 修改对应 Nginx 配置中的 `server xxx:8080`，`docker exec nginx nginx -s reload` 切换流量。
5. 等待后停止旧实例。实现**零停机发版**。

```bash
# 发布某个后端模块示例
cd /root/wg-app/saas && ./blue_green_deploy.sh
# pm 模块还提供 rollback.sh 回滚脚本
```
> 注意脚本顶部 `NGINX_CONF_PATH` 需指向该模块对应的 Nginx upstream 配置文件。

### 前端
直接替换 `/opt/html/<站点>/` 下的静态文件即可（HTML 设为 no-cache，立即生效）。

### 中间件
```bash
cd /root/wg-app && docker compose up -d   # 启动 nginx/mysql/redis/minio
```

## 八、常用运维命令

```bash
# 查看所有容器状态
docker ps -a

# 查看某后端日志
docker logs -f saas-1
tail -f /root/wg-app/saas/data/app/logs/*.log

# 进入数据库
docker exec -it wg-mysql mysql -uroot -p123456

# 重新加载 Nginx（改完 vhost 后）
docker exec nginx nginx -t && docker exec nginx nginx -s reload

# 重启某个模块
cd /root/wg-app/saas && docker compose restart saas-1
```

## 九、安全提醒

> 以下为巡检发现的风险点，建议整改：

1. **凭据弱口令**：MySQL/Redis/MinIO 均为 `123456`/`12345678`，且明文存于 `.env`。
2. **`.env` 含大量第三方密钥**（微信、百度、支付、短信、快递100 等）以明文存储，建议改用密钥管理并轮换。
3. **Docker API 对外暴露**：`dockerd` 监听 `0.0.0.0:2375`（无 TLS），存在被接管风险，建议关闭或限制。
4. **CORS 全开**：多数站点 `Access-Control-Allow-Origin: *`，生产环境建议收敛到具体域名。
5. 生产/测试混用 `test` profile，建议区分环境。
